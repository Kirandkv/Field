# Evaluation Methodology — FieldForge Docs (Slice 1)

Every metric below follows the same rule: **definition → computation → dataset version → baseline
→ current result → acceptance threshold → limitations**. Values not yet measured are written as
`TBD` — never estimated or invented. Current results live in `evals/reports/`, generated by
`scripts/run_eval.py`, which anyone can re-run.

## RAG metrics (slice 1 scope)

| Metric | Definition | Computation | Limitations |
|---|---|---|---|
| Recall@k | Fraction of eval cases where a chunk from the gold source page is in the top-k retrieved | Exact chunk/page-id match against `EvaluationCase.gold_chunk_ids` | Corpus is small (single digits of docs); not representative of large-corpus recall |
| MRR | Mean reciprocal rank of the first gold chunk across all cases | Standard MRR over BM25-ranked results | Same corpus-size caveat |
| Citation correctness | Fraction of answers whose cited chunk ids both exist and actually support the claim they're attached to | Automated: cited id resolves in index. Manual spot-check for "actually supports" until an LLM-judge is wired (M2) | Automated check alone cannot fully verify semantic support — documented as a slice-1 limitation |
| Refusal accuracy | Fraction of intentionally-unanswerable eval cases where the system correctly refuses instead of guessing | Exact match against `EvaluationCase.expect_refusal` | Small unanswerable subset in slice 1 |
| p50 / p95 latency | Wall-clock time for `/query` end-to-end | Measured directly during eval run, not simulated | Local machine only, not representative of production load |

Not yet in scope for slice 1 (require dense/hybrid retrieval or a live LLM adapter, both
deferred to M2 per [ADR 0001](adr/0001-monorepo-vertical-slice.md)): Precision@k beyond gold-page
match, nDCG, context precision/recall (Ragas), answer relevance, faithfulness (Ragas), multilingual
retrieval quality, OCR-noise robustness, table-QA accuracy, token consumption, estimated cost.
These are listed here, as `TBD`, rather than omitted, so the gap is visible.

## Guardrail metrics (slice 1 scope)

| Metric | Definition | Computation |
|---|---|---|
| Injection block rate | Fraction of adversarial cases (direct + indirect injection) correctly blocked/flagged | Automated against `evals/datasets/guardrails_docs_v1.jsonl` |
| PII/secret detection recall | Fraction of planted PII/secret fixtures correctly flagged | Automated regex match against known fixtures |
| False-positive rate | Fraction of benign cases incorrectly blocked | Automated against benign control cases in the same dataset |

## Acceptance thresholds

Not set until the first baseline run produces real numbers (`evals/baselines/docs_v1_baseline.json`).
Setting a threshold before a baseline exists would be a guess, not an engineering decision — this
repo does not do that. Thresholds are added in the PR that first meets them, with the baseline
number cited as justification.

## Reproducing results

```bash
python -m venv .venv && .venv\Scripts\activate  # Windows
pip install -e ".[dev]"
python scripts/run_eval.py --dataset evals/datasets/docs_qa_v1.jsonl --out evals/reports/
```
